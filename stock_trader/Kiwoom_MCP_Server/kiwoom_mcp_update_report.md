# 키움증권 REST API MCP 서버 업데이트 보고서

## 1. 개요
'키움 REST API 문서.pdf'의 최신 사양을 바탕으로 `kiwoom_mcp.py`의 오류를 수정하고, 신규 데이터 조회 기능을 추가하였습니다.

## 2. 주요 수정 내역

### 2.1 주식 주문 기능 (place_order)
- **오류 수정**: 기존에 혼용되던 `api-id`를 PDF 사양에 맞춰 정확히 분리했습니다.
  - **매수**: `kt10000` (PDF p.423)
  - **매도**: `kt10001` (PDF p.425)
- **파라미터 확장**: `ord_tp` (주문구분) 파라미터를 추가하여 지정가(00), 시장가(03) 등을 선택할 수 있게 했습니다.
- **필드 정규화**: PDF 명세에 없는 `buy_sell_tp`를 제거하고, `unit_tp` (단위구분) 필드를 기본값 '1'(단주)로 추가했습니다.

### 2.2 계좌 및 잔고 조회
- **계좌 목록 조회 추가**: `ka00001` API를 연동하여 사용자의 모든 계좌 번호를 확인할 수 있는 `get_account_list` 도구를 추가했습니다.
- **잔고 조회 옵션화**: `get_balance`에 `qry_tp` 파라미터를 추가하여 '합산(1)' 또는 '개별종목(2)' 조회가 가능하도록 개선했습니다.

## 3. 신규 추가된 도구 (MCP Tools)

| 도구명 | API ID | 설명 |
| :--- | :--- | :--- |
| `get_account_list` | `ka00001` | 사용 가능한 계좌 번호 목록 조회 |
| `get_stock_list` | `ka10099` | 시장별(코스피/코스닥) 전체 종목 리스트 조회 |
| `get_daily_chart` | `ka10081` | 특정 종목의 기준일자별 일봉 데이터 조회 |

## 4. 기술적 개선 사항
- `KiwoomApiManager._request` 메서드에서 POST/GET 요청 판별 로직을 강화하여 `body`가 빈 객체(`{}`)인 경우에도 올바르게 POST 요청을 보내도록 수정했습니다.
- 각 도구에 상세한 docstring을 추가하여 MCP 클라이언트에서 기능을 쉽게 파악할 수 있도록 했습니다.
